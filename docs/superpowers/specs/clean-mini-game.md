# 清理大师小游戏 — 实现规格

## 概述

微信小游戏（Canvas 2D），玩法：玩家选择正确的清洁工具，手指拖动擦除鞋子表面污垢，分阶段完成清洁任务。每阶段擦净 90% 自动进入下一阶段，全部完成后显示胜利弹窗。

## 技术方案

Canvas 2D 双层渲染 + `destination-out` 擦除：

| 层级 | 内容 | 说明 |
|------|------|------|
| 底层 | 干净鞋子图片 | `images/shoe_clean.png` |
| 顶层 | 污垢纹理（离屏 Canvas） | 程序生成，按鞋子轮廓裁剪 |

- 手指划过时在污垢层用圆形笔刷挖洞，露出底层干净鞋面
- 每 500ms 采样一次 `getImageData` 统计擦净比例
- 污垢层通过 `destination-in` 合成鞋子蒙版，确保脏污只出现在鞋子区域内

## 游戏流程

```
启动 → 加载鞋子图片 → 生成第1阶段污垢层
  → 玩家选择工具 → 手指擦除
  → 擦净 ≥ 90% → 进入下一阶段（新污垢层）
  → 所有阶段完成 → 显示胜利弹窗 → 可重玩
```

## 阶段系统（stages.js）

游戏分多个清洁阶段，顺序执行：

| 步骤 | 名称 | 必需工具 | 过关阈值 | 污垢颜色 | 笔刷半径 |
|------|------|----------|----------|----------|----------|
| 1 | 刷灰 | 刷子 | 90% | 浅灰 `#9a9a9a` α=0.85 | 45 |
| 2 | 擦净 | 抹布 | 90% | 深棕 `#4a3020` α=0.7 | 25 |

每个阶段配置一个独立 `DirtLayer` 实例，切换阶段时销毁旧层、新建新层（保留鞋子蒙版）。

### 阶段配置字段

```js
{
  name: '第1步 刷灰',          // 标题栏显示
  tool: '刷子',                 // 必须选中此工具才能擦除
  passThreshold: 0.9,           // 擦净比例达到此值进入下一阶段
  layer: {
    baseColor: '#9a9a9a',       // 污垢底色
    baseAlpha: 0.85,            // 底色透明度
    noiseCount: 300,            // 噪点数量（增加污垢质感）
    noiseColorRange: { ... },   // 噪点 RGB 随机区间
    brushRadius: 45             // 该阶段擦除笔刷半径
  }
}
```

## 核心模块

### DirtLayer.js — 污垢层

- 持有一个独立离屏 Canvas（与主 Canvas 同尺寸）
- 构造时根据 `rect`（鞋子区域）绘制底色 + 随机噪点
- `setMask(img)`：用 `destination-in` 把鞋子图作为蒙版裁剪污垢，使污垢只出现在鞋子轮廓内；同时重新统计初始不透明像素数作为分母
- `erase(x, y, radius)`：在 (x, y) 处用 `destination-out` 画圆，挖出透明区域
- `getCleanRatio(now)`：每 500ms 调用一次 `getImageData` 统计 alpha ≥ 128 的像素，计算 `1 - 剩余 / 初始`；高频调用直接返回缓存值

### GameScene.js — 主场景

- 构造时计算鞋子区域 `shoeRect`：顶部留 56px 给标题栏，底部留 90px 给工具栏，中间最大化容纳鞋子图（保持原图宽高比，居中）
- 每帧绘制顺序：清屏 → 鞋子底图 → 污垢层 → UI
- 触摸处理：
  - `onTouchStart`：先判断是否点中底部工具按钮，命中则切换 `selectedTool`，否则当作擦除起点
  - `onTouchMove`：若 `selectedTool ≠ stage.tool` 触发"工具不对"提示；否则对每个触点调用 `dirt.erase()`
  - `onTouchEnd`：仅在胜利弹窗中处理"再玩一次"按钮
- 阶段切换：当 `cleanRatio ≥ passThreshold` 时调用 `_advanceStage()`，新建下一阶段污垢层并触发 2 秒切换提示；最终阶段完成后置 `won = true` 显示胜利弹窗

### UI.js — 界面

四个区域：

1. **顶部标题栏**（高 56px，半透明黑底）
   - 当前阶段名 + 步骤序号 `1/2`
   - 进度条：宽 = 屏宽 - 28，高 6px，未到 90% 蓝色，达标后绿色
   - 右上角百分比文字

2. **底部工具栏**（高 90px，半透明黑底）
   - 工具按钮（刷子 / 抹布），圆角矩形 + 简笔图标 + 名称
   - 选中态：蓝底 `#2196F3`
   - 推荐态（当前阶段所需且未选中）：金色脉冲描边
   - `hitTool(x, y)` 返回命中的工具名

3. **临时提示**（屏幕中央，自动淡出）
   - 工具不对：红色 toast，1.5 秒淡出
   - 阶段切换：黑色 toast 显示新阶段名，2 秒淡出

4. **胜利弹窗**（`showWin = true` 时绘制）
   - 半透明遮罩 + 白色圆角卡片
   - "清洁完成！" + "鞋子焕然一新"
   - "再玩一次"按钮（`hitReplay(x, y)` 命中检测）

## 关键技术点

### Canvas 合成模式

| 模式 | 用途 |
|------|------|
| `source-over`（默认） | 正常叠加绘制 |
| `destination-in` | 蒙版裁剪：保留与新图重叠的部分（鞋形污垢） |
| `destination-out` | 擦除：在重叠处挖洞（手指擦除） |

每次合成操作后必须改回 `source-over`，避免污染后续绘制。

### 像素统计性能优化

- `getImageData` 是同步操作，对全屏画布每帧调用会卡顿
- 节流策略：500ms 内重复调用直接返回缓存比例
- 仅扫描 `rect` 区域（鞋子矩形）而非整个画布
- 仅检查 alpha 通道（`i += 4` 跳过 RGB），减少 75% 内存读取

### 鞋子图按比例适配

`_computeShoeRect(img)` 比较图片宽高比与可用区域宽高比：
- 图片更宽 → 以宽度为准，按比例缩高
- 图片更高 → 以高度为准，按比例缩宽
- 在可用区域内居中

### 多阶段切换

切换时新建 `DirtLayer` 实例而非清空旧层，原因：
- 不同阶段污垢颜色、密度、笔刷半径都不同
- 新建后立即调用 `setMask(_shoeImg)` 重新裁剪鞋形
- `_initialDirt` 重置为新层的初始像素数，进度条从 0 开始
